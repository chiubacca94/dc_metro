What "@tailwindcss/postcss" means
Conceptually, @tailwindcss/postcss would be the Tailwind CSS plugin for PostCSS — i.e., the code PostCSS runs to transform Tailwind directives (like @tailwind, @apply, @layer) into generated CSS utilities.
In practice you normally use the tailwindcss package itself as the PostCSS plugin. For example, PostCSS configs use:
require('tailwindcss') or the shorthand key tailwindcss as the plugin entry.
Historically there have been related packages for compatibility (for example @tailwindcss/postcss7-compat), but there isn't a common public package named exactly @tailwindcss/postcss that you need to install — tailwindcss is the package that provides the PostCSS plugin.

How Tailwind + PostCSS fit into your build
PostCSS runs as part of the CSS build pipeline (via a bundler or tooling like react-scripts).
The Tailwind PostCSS plugin processes your CSS:
Expands @tailwind base/components/utilities into the framework CSS.
Resolves @apply rules.
Removes unused styles if configured with Purge/Content settings (in production builds).
After Tailwind runs, Autoprefixer usually runs to add vendor prefixes.
